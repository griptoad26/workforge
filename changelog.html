Changelog

Every release of XKG Desktop, in order.

v0.1.0 · Initial release

First public build

2026-06-21

Desktop (Tauri 2 + Svelte 5 + Vite 5)

  • Global keyboard shortcut — Ctrl+Shift+X (or Cmd+Shift+X on macOS), configurable via XKG_GLOBAL_SHORTCUT env var. Opens a 480×280 floating capture window at the cursor.
  • Local SQLite with FTS5 — full-text search across all conversations, messages, providers, and sync logs. Schema in ~/.local/share/xkg-desktop/xkg-desktop.db.
  • System tray with settings panel (720×520) for hub URL, hotkey, and autostart.
  • Autostart on login via tauri-plugin-autostart.
  • Cluster-hub sync — captures POST to XKG_HUB_URL/api/import (default http://127.0.0.1:8090) with {text, title, source, tags}.
  • Eight build targets — Linux (.deb, .rpm, AppImage), Windows (NSIS .exe, MSI), macOS (Intel .dmg + Apple Silicon .dmg).

Mobile (Flutter)

  • Android APK (22 MB) — sideload-ready, minSdk 21+.
  • Linux bundle (9 MB) — single binary, runs from tar xzf.
  • Windows bundle (44 MB) — ZIP with all required DLLs.
  • JSONL bridge reader — picks up captures from ~/.local/share/xkg-mobile/from_desktop.jsonl on launch.

Cluster-hub (Python)

  • POST /api/import — accepts captures, returns {ok, id, total_imports}.
  • Appends to JSONL bridge file after every successful import.
  • Backed by xkg-user-imports.json for master storage.

Cross-device

  • Capture on any device → cluster-hub → JSONL → mobile Hive box → full-text searchable.
  • Verified end-to-end: POST /api/import from desktop, see it on mobile after restart.

Known limitations

  • Windows MSI built with manual WiX (not the Tauri-templated bundle). Installs correctly but lacks some Tauri metadata (no Start Menu shortcut).
  • No code signing yet (Windows + macOS will show "unverified publisher" warnings). Coming in v0.2.0.
  • Conflict resolution is last-write-wins. Vector clocks + CRDT planned for v0.3.0.
