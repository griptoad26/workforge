🔄 How Data Flows

1️⃣ Tasks Flow (Push to Hub)

win-20o3 creates task → Cluster Hub (8090) ← All NUCs can see task

2️⃣ Discord History (Query Hub)

EVO-X2 runs Discrawl ↓ SQLite DB ↑ win-20o3 queries → Hub API

3️⃣ Files Sync

Push to Hub → Hub Files ← Pull from Hub

4️⃣ Oracle (Local LLM)

User Query → Mistral on EVO-X2 → Review/Enhance → Better Response

✨ Benefits of This Architecture

Why NOT Sync Everything Everywhere

  • ❌ Bandwidth waste - 381 messages × 7 NUCs = 2,667 copies
  • ❌ Storage waste - SQLite DB on every machine
  • ❌ Security risk - Private messages on all machines
  • ❌ Sync complexity - 7 machines × Discord API rate limits

Why Hub + Query Works Better

  • ✅ Single source - One Discrawl instance
  • ✅ On-demand - Query when needed
  • ✅ Secure - Data stays on EVO-X2
  • ✅ Simple - One sync job to maintain

🚀 How to Connect a New NUC

  1. Download OCMI modules from Hub: 
    curl -o ocmi_settings.py http://100.112.11.35:8090/api/files/ocmi_settings.py
curl -o sync_cluster_hub.py http://100.112.11.35:8090/api/files/sync_cluster_hub.py
  2. Set OCMI config: 
    {
  "ocmi": {
    "sync": {
      "enabled": true,
      "provider": "cluster-hub",
      "cluster_hub_url": "http://100.112.11.35:8090"
    }
  }
}
  3. Now the NUC can:
    • ✅ Push/pull tasks to Hub
    • ✅ Query Discord history via Hub API
    • ✅ Use Oracle (Mistral on EVO-X2)
