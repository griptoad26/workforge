OpenClaw Memory Infrastructure - Installation Guide

Version 0.4 • March 2026

⚡ Quick Install

One-command setup for new machines:

curl -sL https://github.com/griptoad26/OCMI---Memory-Infrastructure/archive/refs/heads/main.zip | unzip - && cd OCMI---Memory-Infrastructure && bash setup.sh

🤔 What is OCMI?

OCMI (OpenClaw Memory Infrastructure) gives your AI agent:

📝
Memory System - Daily notes + long-term memory that persists between sessions
📋
Task Extraction - Automatically finds tasks in your conversations
☁️
Cloud Sync - Optional sync to Cluster Hub, OneDrive, or Nextcloud
🔮
Oracle Mode - Local LLM for reviewing/enhancing responses

🚀 Installation

Step 1: Clone the Repository

# Clone OCMI to your machine
git clone https://github.com/griptoad26/OCMI---Memory-Infrastructure.git
cd OCMI---Memory-Infrastructure

Step 2: Initialize Settings

# Run the setup script
python3 setup.sh

# Or initialize manually
cd memory
python3 -c "import json; open('../.memory/config.json','w').write(json.dumps({
    'version': '0.4',
    'memory': {'load_longterm_in_guild': False},
    'sync': {'enabled': False}
}))"

Step 3: Verify Installation

# Check settings
python3 memory/ocmi_cli.py show
✅ OCMI is installed! Your agent now has memory capabilities.

⚙️ Configuration

Enable Memory in Discord

cd memory
python3 ocmi_cli.py set ocmi.memory.load_longterm_in_guild true
Why? This lets the agent read MEMORY.md in Discord group chats, giving it context about your past conversations.

Enable Sync (Optional)

# Enable Cluster Hub sync
python3 ocmi_cli.py set ocmi.sync.enabled true
python3 ocmi_cli.py set ocmi.sync.provider cluster-hub
python3 ocmi_cli.py set ocmi.sync.cluster_hub_url http://100.112.11.35:8090

Enable Oracle (Optional)

# Use local Mistral LLM for response review
python3 ocmi_cli.py set ocmi.oracle.enabled true
python3 ocmi_cli.py set ocmi.oracle.url http://your-oracle:8081

🔧 Troubleshooting

Module Not Found Error

# Make sure you're in the right directory
cd ~/.openclaw/workspace/OCMI---Memory-Infrastructure/memory
python3 ocmi_cli.py show

Memory Not Loading

# Check if memory is enabled
python3 ocmi_cli.py get ocmi.memory.load_longterm_in_guild

# Enable it
python3 ocmi_cli.py set ocmi.memory.load_longterm_in_guild true

Tasks Not Being Created

Task extraction requires the hook to be wired into your session. See:

https://github.com/griptoad26/OCMI---Memory-Infrastructure
