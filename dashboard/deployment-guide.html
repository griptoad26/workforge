🪟 Windows Native Installation

Run in: PowerShell (Administrator)

Step 1: Check Prerequisites

# Check Python (should be installed)
python --version

# Check Tailscale
tailscale status

Step 2: Install Python (if needed)

# Using winget
winget install Python.Python.3.12

# OR download from https://python.org

Step 3: Install Tailscale

# Install Tailscale in PowerShell (Admin), NOT WSL!
winget install tailscale

# Start Tailscale
Start-Service Tailscale

# Verify
tailscale status
⚠️ IMPORTANT: Tailscale must be installed on Windows side, NOT in WSL. WSL will use Windows' Tailscale automatically.

Step 4: Create Directory

New-Item -ItemType Directory -Path C:\OpenClaw -Force
cd C:\OpenClaw

Step 5: Download OCMI Modules

$env:CLUSTER_HUB_URL = "http://100.112.11.35:8090"

Invoke-WebRequest -Uri "$env:CLUSTER_HUB_URL/api/files/ocmi_settings.py" -OutFile ocmi_settings.py
Invoke-WebRequest -Uri "$env:CLUSTER_HUB_URL/api/files/sync_cluster_hub.py" -OutFile sync_cluster_hub.py
Invoke-WebRequest -Uri "$env:CLUSTER_HUB_URL/api/files/cross_session.py" -OutFile cross_session.py
# Register this machine with Cluster Hub $hostname = $env:COMPUTERNAME Invoke-WebRequest -Uri "http://100.112.11.35:8090/api/nodes/register" -Method POST -ContentType "application/json" -Body "{`"hostname`":`"$hostname`",`"os`":`"Windows`",`"capabilities`":[`"ocmi`"]}"

Step 6: Set Environment Variables

Understanding System-wide vs Session:
MethodScopePersistsRequires
"Machine"All users, all processesAfter rebootAdmin
"User"Current user onlyAfter rebootNone
$env:Current session onlyNoNone

Option A: Current Session Only (Try First)

# This works for this PowerShell session only
# Easy to test, no admin needed
$env:CLUSTER_HUB_URL = "http://100.112.11.35:8090"
$env:OCMI_SYNC_ENABLED = "true"

# Test it
$env:CLUSTER_HUB_URL

# Warning: Closes when you close PowerShell

Option B: Machine-wide (Persistent)

# This affects ALL users, forever (until changed)
# Requires PowerShell as Administrator
[System.Environment]::SetEnvironmentVariable("CLUSTER_HUB_URL", "http://100.112.11.35:8090", "Machine")
[System.Environment]::SetEnvironmentVariable("OCMI_SYNC_ENABLED", "true", "Machine")

# Verify
[System.Environment]::GetEnvironmentVariable("CLUSTER_HUB_URL", "Machine")

# This takes effect for NEW PowerShell sessions

Option C: User-only (Persistent)

# Current user only, persistent after reboot
# No admin required
[System.Environment]::SetEnvironmentVariable("CLUSTER_HUB_URL", "http://100.112.11.35:8090", "User")
[System.Environment]::SetEnvironmentVariable("OCMI_SYNC_ENABLED", "true", "User")

# Verify  
[System.Environment]::GetEnvironmentVariable("CLUSTER_HUB_URL", "User")
Recommendation: Start with Option A ($env:) to test. Once confirmed working, use Option B ("Machine") for permanent setup.

Step 7: Tell Your Agent About OCMI ⚡

Do not skip this! Your agent needs to know OCMI is installed. 
I have installed OCMI v0.4. 

https://github.com/griptoad26/OCMI---Memory-Infrastructure

Configure:
- python ocmi_cli.py set ocmi.memory.load_longterm_in_guild true
- python ocmi_cli.py set ocmi.sync.enabled true

Remember: OCMI v0.4 is active.

Step 8: Configure Oracle (Optional)

# To use EVO-X2's Mistral as Oracle:
# Edit ocmi_settings.json or set:
$env:ORACLE_ENABLED = "true"
$env:ORACLE_URL = "http://100.112.11.35:8081"
✅ Done! Your Windows machine is now connected to Cluster Hub.

🔮 Step 7: Configure Oracle (Optional)

What is Oracle? Uses local Mistral to review/enhance cloud LLM responses. Points to Mistral running on EVO-X2.

Option A: Use EVO-X2's Mistral (Recommended)

# Point to Mistral on EVO-X2
# Current setting (on EVO-X2):
# oracle.local_model_url = "http://localhost:8081"

# From Windows, use EVO-X2's Tailscale IP:
$env:ORACLE_URL = "http://100.112.11.35:8081"

# Enable Oracle
$env:ORACLE_ENABLED = "true"

# Test it
curl http://100.112.11.35:8081/v1/models

Option B: No Oracle (Default)

# Just skip this step
# Oracle is optional - works without it

How Oracle Works

User Query → Cloud LLM → Oracle (Mistral) reviews → Better Response
              ↑
         Local model on EVO-X2 can also enhance queries before sending
No local GPU needed! Oracle uses EVO-X2's Mistral over the network.

🐧 WSL Installation (Windows Subsystem for Linux)

⚠️ Dual Installation Required

WSL requires TWO separate installations:

WhereWhatTerminal
WindowsTailscalePowerShell (Admin)
WSLOCMI modulesUbuntu app

Part 1: Windows Side (PowerShell Admin)

Run in: PowerShell (Administrator)
# Install Tailscale on Windows
winget install tailscale
Start-Service Tailscale
⚠️ DO NOT install Tailscale in WSL! It will conflict with Windows Tailscale.

Part 2: WSL Side (Ubuntu Terminal)

Run in: Ubuntu App (not PowerShell!)
# Update system
sudo apt update && sudo apt upgrade -y

# Install Python
sudo apt install -y python3 python3-pip python3-venv

# Create OpenClaw directory
mkdir -p ~/.openclaw/memory
cd ~/.openclaw/memory

# Download OCMI modules
export CLUSTER_HUB_URL="http://100.112.11.35:8090"
curl -o ocmi_settings.py "$CLUSTER_HUB_URL/api/files/ocmi_settings.py"
curl -o sync_cluster_hub.py "$CLUSTER_HUB_URL/api/files/sync_cluster_hub.py"
curl -o ocmi_cli.py "$CLUSTER_HUB_URL/api/files/ocmi_cli.py"
curl -o session_integration.py "$CLUSTER_HUB_URL/api/files/session_integration.py"
curl -o cross_session.py "$CLUSTER_HUB_URL/api/files/cross_session.py"

# Register this machine with Cluster Hub
HOSTNAME=$(hostname)
curl -X POST "http://100.112.11.35:8090/api/nodes/register" \\
  -H "Content-Type: application/json" \\
  -d "{\"hostname\":\"$HOSTNAME\",\"os\":\"Linux\",\"capabilities\":[\"ocmi\"]}"

# Install dependencies
pip3 install requests flask

Part 3: Environment Variables

# Add to ~/.bashrc for persistence (every new terminal)
echo 'export CLUSTER_HUB_URL="http://100.112.11.35:8090"' >> ~/.bashrc
echo 'export OCMI_SYNC_ENABLED="true"' >> ~/.bashrc

# Apply to CURRENT terminal (doesn't persist)
source ~/.bashrc

# Verify
echo $CLUSTER_HUB_URL

Part 4: Oracle Configuration

# Point to EVO-X2's Mistral
echo 'export ORACLE_ENABLED="true"' >> ~/.bashrc
echo 'export ORACLE_URL="http://100.112.11.35:8081"' >> ~/.bashrc
source ~/.bashrc
✅ Done! WSL is connected to Cluster Hub. WSL uses Windows' network automatically.

Step 5: Create Systemd Service (Optional)

sudo nano /etc/systemd/system/openclaw.service

# Paste:
[Unit]
Description=OpenClaw Agent
After=network.target

[Service]
Type=simple
User=$USER
WorkingDirectory=%h/openclaw
ExecStart=/usr/bin/python3 -c "from sync_cluster_hub import get_sync; get_sync().register(); import time; time.sleep(999999)"
Restart=always
Environment=CLUSTER_HUB_URL=http://100.112.11.35:8090

[Install]
WantedBy=multi-user.target

# Enable
sudo systemctl daemon-reload
sudo systemctl enable openclaw
sudo systemctl start openclaw

🗑️ Removing Cluster Hub Connection

If you want to completely remove the Cluster Hub connection from a machine:

On the Machine (Local)

# Stop any sync services
pkill -f sync_cluster_hub

# Remove OCMI files
rm -f ~/.openclaw/memory/ocmi_settings.py
rm -f ~/.openclaw/memory/sync_cluster_hub.py
rm -f ~/.openclaw/memory/cross_session.py

# Remove environment variables
sed -i "/CLUSTER_HUB_URL/d" ~/.bashrc
sed -i "/OCMI_SYNC_ENABLED/d" ~/.bashrc

# Remove OpenClaw sync config
rm -f ~/.openclaw/memory/.memory/config.json 2>/dev/null

From Cluster Hub (Server)

# On EVO-X2, remove the node registration:
curl -X DELETE http://100.112.11.35:8090/api/nodes/NODE_NAME

# Example for t1-nuc:
curl -X DELETE http://100.112.11.35:8090/api/nodes/t1-nuc
Note: The node will still appear in Tailscale, but will be removed from the Cluster Hub dashboard and no longer receive task/file updates.

⚙️ OCMI CLI Commands

After installation, use the OCMI CLI to configure settings:

# Initialize OCMI settings (first time)
python3 ocmi_cli.py init

# Show all current settings
python3 ocmi_cli.py show

# Get a specific setting
python3 ocmi_cli.py get ocmi.memory.load_longterm_in_guild
python3 ocmi_cli.py get ocmi.sync.enabled

# Enable/disable features
python3 ocmi_cli.py set ocmi.memory.load_longterm_in_guild true
python3 ocmi_cli.py set ocmi.sync.enabled true
python3 ocmi_cli.py set ocmi.sync.provider cluster-hub
python3 ocmi_cli.py set ocmi.oracle.enabled true

Recommended Settings

# Enable memory in Discord channels (for persistent context)
python3 ocmi_cli.py set ocmi.memory.load_longterm_in_guild true

# Enable Cluster Hub sync (to share tasks between machines)
python3 ocmi_cli.py set ocmi.sync.enabled true
python3 ocmi_cli.py set ocmi.sync.provider cluster-hub
python3 ocmi_cli.py set ocmi.sync.cluster_hub_url http://100.112.11.35:8090
